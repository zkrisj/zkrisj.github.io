Proxy 中的 this 指向

746字 2022-09-29
4 min read
Scroll Down

介绍

Proxy 和 Reflect 内置对象从 ECMAScript 2015 开始支持，允许拦截并定义自定义行为（包括 apply, construct, defineProperty, deleteProperty, get, getOwnPropertyDescriptor, getPrototypeOf, has, isExtensible, ownKeys, preventExtensions, set, setPrototypeOf）。

语法

const p = new Proxy(target, { ...traps })
  • traps
    • 被代理对象上的自定义行为。包含捕获器（操作发生时所对应的处理函数）的方法列表。
  • target
    • 被代理的目标对象。
const handler = {
  get(target, name, receiver) {
    // 如果对象中包含该属性，返回该属性，否则返回 not exists
    return name in target ? Reflect.get(...arguments) : 'not exists';
  },
  set(target, name, value, receiver) {
    // 如果属性为 age，判断值的类型是否为 number 和大于 0
    if (name === 'age') {
      if (!Number.isNaN(parseInt(value))) {
        target[name] = value > 0 ? value : 0;
        return true;
      } else throw 'invalid age';
    } else return Reflect.set(...arguments);
  },
};

const p = new Proxy({}, handler);
p.a = 1;
console.log(p.a, p.b); // 1, not exists
p.age = 12;
console.log(p.age); // 12
p.age = -12;
console.log(p.age); // 0
p.age = 'a';
console.log(p.age); // Uncaught invalid age

使 Proxy 起作用，必须对 proxy对象\color{red}{proxy 对象}proxy对象 进行操作，而不是原对象。

this 绑定

在 Proxy 代理的情况下，原对象内部的 this 会指向 Proxy 代理。

const target = {
  m () {
    return this === proxy;
  }
};
const proxy = new Proxy(target, {});

console.log(target.m()); // false
console.log(proxy.m());  // true

下面在新建 Person 对象时，存储在外部 WeakMap 对象上面的键是 Person 对象 jane，所以 _name.get(proxy) 返回 undefined。通过 proxy.name 访问时，this 指向 proxy，也返回 undefined。

const _name = new WeakMap();

class Person {
  constructor(name) {
    _name.set(this, name);
  }
  get name() {
    return _name.get(this);
  }
}

const jane = new Person('Jane');
console.log(jane.name); // 'Jane'
console.log(_name.get(jane)); // 'Jane'

const proxy = new Proxy(jane, {});
console.log(proxy.name); // undefined
console.log(_name.get(proxy)); // undefined

handler 中的 this

handler 拦截函数内部的 this，指向的是 handler 对象本身。

const handler = {
  get() {
    console.log(this === handler);
  }
};
new Proxy({}, handler).foo; // true

原生对象

有些原生对象的内部属性，只有通过正确的 this 才能获取。

const target = new Date();
new Proxy(target, {}).getDate(); // TypeError: this is not a Date object

const handler = {
  get(target, prop) {
    const val = target[prop];
    return typeof val === 'function' ? val.bind(target) : Reflect.get(...arguments);
  }
};
const proxy = new Proxy(target, handler);
console.log(proxy.getDate());

receiver

get 和 set 捕获器函数的最后一个参数都是一个 receiver 对象，指向当前 Proxy 对象。

const proxy = new Proxy({}, {
  get(target, key, receiver) {
    return receiver;
  }
});
proxy.getReceiver === proxy // true

但是，Proxy 对象作为其他对象的原型被继承时，receiver 会指向被继承的对象。

let obj;
const proxy = new Proxy({
  foo: 3
}, {
  get(target, prop, receiver) {
    console.log(receiver === proxy, receiver === obj);
    return target[prop];
  }
});
console.log(proxy.foo); // true false 3
obj = Object.create(proxy);
console.log(obj.foo); // false true 3

getter

在捕获器方法中，如果遇到 target 中有定义的 getter 属性，则 getter 中的 this 指向调用 target 提供的 this 值，即不会改变指向，但可以手动将其指向 receiver。例如下面在原对象中定义的 getter 属性 info。

const obj = {
  foo: 1
};
const proxy = new Proxy({
  foo: 3,
  get info() {
    return this.foo;
  }
}, {
  get(target, prop, receiver) {
    console.log(receiver === obj); // true
    console.log(target[prop]); // 3
    console.log(Reflect.get(target, prop)); // 3
    return Reflect.get(...arguments); // Reflect.get(target, prop, receiver)
  }
});
Object.setPrototypeOf(obj, proxy);
console.log(obj.info); // 1
